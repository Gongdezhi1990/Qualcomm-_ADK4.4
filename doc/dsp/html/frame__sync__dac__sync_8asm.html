Detailed Description

The CBOPS library copies the minimum of the amount of data in the input buffers and the amount of space in the output buffers. However, for the DAC we want to maintain the Minimum amount of data in the Port to minimize latency. On the other hand we need to maintain sufficient data in the Port to prevent a wrap condition where the read pointer (limit) passes the write pointer (offset). Also, to maintain syncronization with the ADC/DAC and the rest of the system we need to drop samples at the DAC when they are inserted in the ADC and visa versa.

DAC Synchronization operator

The $frame_sync.dac_sync_op and $frame_sync.dac_wrap_op operators work in conjunction to provide this functionality.

The $frame_sync.dac_wrap_op operator must be the last operator in the chain and the $frame_sync.dac_sync_op operator should be inserted just before the operator that actually write to the DAC. If a sidetone mix operation is included in the chain the $frame_sync.dac_sync_op operator should precede it as well
