Detailed Description

[SDK Folder]/src/kalimba/spdif/latency_measurement.asm

DESCRIPTION

Measuring the latency of the spdif for both coded and pcm input data type

For PCM data type, the measured latency will be the total buffering data including those in ports for input and output interfaces .

For coded data, the latency is updated once per IEC61937 frame, the measured latency will be difference between the arrival time of a packet(when first word of packet is seen in the input port) and the time the the (first sample of the) packet leaves for the output interface.
