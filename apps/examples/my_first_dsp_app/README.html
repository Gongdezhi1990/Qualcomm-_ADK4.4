Description

This example application consists of both DSP and VM software to implement a simple audio example using the Kalimba DSP. The application routes audio from the development board mic/line inputs to the headset output.

Note: No Bluetooth functionality is required or invoked in this example.

This ReadMe is not intended to describe procedures that are covered in the xIDE user guide such as building projects in xIDE etc. For the detailed information on how to run the my_first_dsp_app, please refer to the application note CS-101420-AN.

Prerequisites

Hardware requirements

This application requires:

  • A BlueCore Kalimba DSP development board with audio output support (e.g. CNS10001 development board or H13179v2 development board)
  • A headset
  • A host PC system with SPI data connection

To download the application to the BlueCore chip, an SPI connection is required between the development hardware and the PC hosting xIDE.

Software requirements

No additional software is required.

Quick Start

  1. Open the my_first_dsp_app example workspace (my_first_dsp_app.xiw) in xIDE.
  2. Select the kalimba - 'my_first_dsp_app_kalimba' project from the drop-down field in the toolbar in xIDE.
  3. Build (F7) the application from within xIDE.
  4. Alternatively, right-click on the kalimba - 'my_first_dsp_app_kalimba' project in the Files pane on the left hand side and select Build.
  5. Select the vm - my_first_dsp_app project from the drop-down field, Build (F7) and then Run (F5) the application.
  6. The application should now play audio received at the mic/line inputs on the headset.

Controlling the Application

Once executed the application runs until stopped, reset or powered down. This is a simple application and no control features are provided.

Configuration Details

No specific application configuration is required.

PS Keys

Default PSKeys can be used.

Please use the 'Describe' feature in the supplied PSTool application for more details about PS Keys.
